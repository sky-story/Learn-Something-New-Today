Introduction

GitHub is a cloud-based platform that hosts Git repositories online. It provides tools for collaboration, code review, project management, and community contributions.

  • Store your code in the cloud.
  • Share your repositories with others.
  • Use features like pull requests, issues, and actions (CI/CD).

Setting Up Git & GitHub

  1. Install Git: git-scm.com
  2. Create a GitHub account: github.com/join
  3. Set up Git with GitHub and create your first repository
bash
git config --global user.name "Your Name" git config --global user.email "you@example.com" # Verify your configuration git config --global --list # Create your first repository mkdir my-first-repo cd my-first-repo git init

Creating a Local Git Repository

Start by creating a new project folder and initializing it as a Git repository. This creates a local version control system for your project.

terminal
# Create a new project folder mkdir your-project-folder cd your-project-folder # Initialize Git repository git init # Add your first file echo "# My Project" > README.md git add . git commit -m "Initial commit" # Check repository status git status

Once your local repository is ready, connect it to GitHub and push your code to the cloud. Make sure to create an empty repository on GitHub first.

terminal
# Add GitHub remote repository git remote add origin https://github.com/your-username/your-repo.git # Set main branch git branch -M main # Push to GitHub for the first time git push -u origin main # For subsequent pushes git add . git commit -m "Your commit message" git push

Uploading to GitHub [First Push]

Common Git Commands

Command Description
git init Initialize a new Git repository in your folder
git status Check current changes and repository status
git add . Stage all files for commit
git add <file> Stage specific file(s) for commit
git commit -m "message" Save a version snapshot with description
git push Upload commits to GitHub
git clone [URL] Download a project from GitHub
git log View commit history and changes

Understanding Branches

Feature Development

Create new features without affecting the main code

git checkout -b new-feature git push origin new-feature

Bug Fixes

Fix issues in isolated branches for testing

git checkout -b bugfix git commit -m "Fixed issue"

Team Collaboration

Work together without conflicts

git pull origin main git merge feature-branch

Experimentation

Try new ideas safely without risk

git branch experiment git checkout experiment

Switch Branches

Move between different versions of your code

git checkout main git switch branch-name
